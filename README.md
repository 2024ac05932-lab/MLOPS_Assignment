## Running each task locally

```bash
# Task 1: download data + EDA
python download_data.py
python eda.py

# Task 2: train + tune + evaluate both models
python modeling.py

# Task 3: MLflow experiment tracking
python mlflow_tracking.py
mlflow ui   # open http://localhost:5000 to view runs

# Task 4: train + package the final pipeline
python train.py
# -> artifacts/heart_disease_model.joblib

# Task 5: run tests
pytest tests/ -v
flake8 train.py src/ tests/

# Task 6: build & run the container locally
cp artifacts/heart_disease_model.joblib heart_disease_pipeline.joblib
docker build -t heart-disease-api:latest .
docker run -d --name heart-api -p 8000:8000 heart-disease-api:latest
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{"age":63,"sex":1,"cp":3,"trestbps":145,"chol":233,"fbs":1,"restecg":0,"thalach":150,"exang":0,"oldpeak":2.3,"slope":3,"ca":0,"thal":6}'
docker stop heart-api && docker rm heart-api

# Task 7: deploy to a local Kubernetes cluster (minikube/kind/Docker Desktop)
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
kubectl get pods
kubectl get svc

# Task 8: monitoring
curl http://localhost:8000/metrics   # Prometheus counter
tail -f api_logs.log                 # request logs
```

## API reference

| Endpoint   | Method | Description                                  |
|------------|--------|-----------------------------------------------|
| `/`        | GET    | Health/welcome message                        |
| `/health`  | GET    | Liveness/readiness check (used by K8s probes) |
| `/predict` | POST   | Returns `{"prediction": 0/1, "confidence": 0-1}` |
| `/metrics` | GET    | Prometheus-format request counter             |

Example `/predict` request body (all 13 features required):
```json
{
  "age": 63, "sex": 1, "cp": 3, "trestbps": 145, "chol": 233,
  "fbs": 1, "restecg": 0, "thalach": 150, "exang": 0,
  "oldpeak": 2.3, "slope": 3, "ca": 0, "thal": 6
}
```

## CI/CD pipeline

`.github/workflows/ci.yml` runs on every push/PR to `main`:

1. **Lint** — flake8 over all source/test files
2. **Test** — pytest against `src/preprocessing.py` and the model pipeline (network-free fixture, no dependency on live UCI access)
3. **Train** — runs `train.py`, uploads the trained model + training logs as workflow artifacts
4. **Containerize** — builds the Docker image, runs it, and curls `/predict` to prove it serves correctly
5. **Deploy** — spins up a disposable `kind` (Kubernetes-in-Docker) cluster inside the runner, applies `deployment.yaml`/`service.yaml`, waits for rollout, and curls `/predict` through the deployed Service — then tears the cluster down

This uses `kind` instead of a persistent cloud cluster because Colab's
sandboxed environment can't run Docker/minikube, and GitHub's `ubuntu-latest`
runners can spin up a real (if ephemeral) Kubernetes cluster for free in
about 30 seconds. See the Actions tab for real `kubectl get pods` / `get svc`
output and a real prediction returned through the deployed Service.

## Models

Two classifiers trained and compared: Logistic Regression and Random Forest,
both tuned via `GridSearchCV` and evaluated with 5-fold cross-validation and
accuracy/precision/recall/F1/ROC-AUC on a held-out test set. See the full
written report for detailed results and discussion.

## Notes on reproducibility

- `ca` and `thal` missing values are imputed with the column mode.
- The target `num` (0-4 severity) is binarized to 0/1 (`num > 0` -> 1).
- The final serving pipeline bundles `StandardScaler` + `RandomForestClassifier`
  in a single `sklearn.Pipeline`, so raw feature order in `/predict` requests
  must match: `age, sex, cp, trestbps, chol, fbs, restecg, thalach, exang, oldpeak, slope, ca, thal`.
